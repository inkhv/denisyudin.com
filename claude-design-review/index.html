Не убийца Figma

Figma падала задолго до релиза. Раздутое IPO, убытки, общий AI-страх по SaaS.

Что такое Claude Design

Отдельная вкладка на claude.ai. Описываешь текстом — Claude строит визуально. Прототипы, лендинги, презентации, баннеры.

Ключевая механика

  • Не генерирует картинки — пишет код: HTML, CSS, React-компоненты
  • Каждый дизайн — работающая веб-страница
  • Результат интерактивный: кнопки кликаются, страница скролится
  • Можно отдать в Claude Code одной командой

Доступ

  • Тарифы Pro, Max, Team, Enterprise
  • Бесплатного доступа нет
  • Только в вебе — не в Claude Code, не в десктопе
  • На Enterprise включается админом вручную

Первая строка системного промпта: «Ты эксперт по дизайну, ты создаёшь дизайн-артефакты через HTML». Claude Design не рисует — он программирует дизайн. Отсюда интерактивность и готовность к передаче в код.

Для кого — и для кого точно нет

Figma и Claude Design конкурируют за разных людей. Это не замена — это соседняя полка.

Claude Design — ваш

  • Фаундер: показать идею инвестору без дизайнера
  • Продукт-менеджер: прототип до разговора с командой
  • Маркетолог: лендинг/баннер за 10 минут
  • Разработчик: визуал перед тем, как начать кодить
  • Стартапер, работающий сам на себя
  • Тот, кто устал от PowerPoint

Claude Design — не ваш

  • Профессиональный дизайнер с настроенной Figma
  • Нужен Pixel Perfect production design
  • Нужна экспортируемая Figma-спецификация
  • Нужна безлимитная генерация
  • Нужны компоненты, auto-layout, плагины, командные библиотеки

Из системного промпта: «Хороший детальный дизайн не начинается с нуля — он опирается на существующий дизайн-контекст. Без контекста результат будет плохим». Это инструмент для тех, кто знает, чего хочет, но не умеет это нарисовать сам.

Режим Prototype

Wireframe vs High Fidelity + Plan Mode

Два режима детализации. Плюс расширенный план-режим, который задаёт 10+ вопросов перед генерацией.

Wireframe

  • Быстрые наброски
  • Структура без визуала
  • Для показа логики, не стиля

High Fidelity

  • Финальный визуал
  • С дизайн-системой
  • Кликабельный прототип

Plan Mode — обязательный

Claude задаёт минимум 10 вопросов. Это цитата из промпта, не пожелание.

  • Продукт и целевая аудитория
  • Главная цель лендинга
  • Язык и тон
  • Что должно быть интерактивным
  • Тарифы, валюта, шрифты
  • Сколько вариантов (1 или 3)

Из системного промпта: «Задавать хорошие вопросы критически важно. Задавай минимум 10 вопросов, а лучше больше». Поэтому ответы на план-режим напрямую определяют качество результата.

Промпт — лайфхаки

Четыре приёма, которые радикально улучшают первый результат и экономят лимит.

1. Негативные указания

Пишите, чего не хотите. Запретить проще, чем потом переделывать.

«без градиентов, без ярких цветов, структура монохромная»

2. Ссылки на реальные продукты

«Вдохновись как Apple» — сработает. «Скопируй интерфейс Apple» — откажется (проверяет email-домен).

3. Пусть Claude Code сам напишет промпт

Откройте Claude Code в терминале: «Напиши детальный промпт для Claude Design. Мне нужен лендинг для X». Получите структурированный промпт на 10–15 строк.

4. Один визуальный запрос — один промпт

Не мегапромпты. «Поменяй шрифт, перекрась фон, подвинь кнопку» — Claude сделает 1–2 вещи, остальное проигнорирует.

Модели: Opus 4.7 — для первого создания и плана. Sonnet 4.6 — для итераций, правок, анимаций. Sonnet справится не хуже, но в разы дешевле по токенам.

Variants и Twix

Две механики итерации, недоступные в обычном Claude Code. Там же — главное правило экономии.

Variants (варианты)

  • В Plan Mode можно заказать 1/2/3 варианта финальной страницы
  • Параллельная генерация разных направлений
  • Выбираете лучшее и докручиваете

Twix (точечные настройки)

  • Слайдер вариантов одного элемента
  • Мгновенный результат без нового промпта
  • Быстрее итераций в Claude Code на порядок

Правило стопа

Смотрите в Updates to-do list сбоку. Если Claude идёт не туда — жмите Stop сразу. Каждая секунда генерации = токены из лимита.

Дешевле остановить на полпути и поправить промпт, чем переделывать готовое.

Slide deck · Other · Templates

Три вкладки, которые недооценивают. А зря — презентации в Claude Design получаются сильнее, чем лендинги.

Slide deck

Speaker notes, анимации, навигация стрелками. Можно сгенерировать видео-ролик из презентации с заданной длиной.

Other

Баннеры, email, одностраничники. Навык генерации баннеров — отличный, только переносы текста требуют правок вручную.

From template

Сохранил один email-шаблон — все будущие рассылки за один промпт. Template — это дизайн-система для структуры, а не для визуала.

CLAUDE.md в дизайн-проекте работает так же, как в Claude Code: постоянные правила для каждой сессии (тёмная тема, шрифт, никаких скруглений). Поведение дизайна, не визуал.

9 секретов системного промпта · 1–5

Из утёкшей 422-строчной инструкции, которую Claude Design получает перед каждой сессией.

01

Claude API внутри дизайна

Артефакт может вызвать Haiku прямо из HTML без API-ключа, в рамках вашей квоты. Кнопка «сгенерировать описание» в прототипе реально работает.

02

7 starter-компонентов

iOS/Android Frame, macOS/Browser Window, DeckStage.js (презентации), Animation.js (таймлайн), Design Canvas (сравнение вариантов).

03

Файлы <1000 строк

Жёсткое правило: «всегда разбивай код на маленькие JSX-файлы». Поэтому экспорт в Claude Code даёт модульный код, а не кашу на 3000 строк.

04

Verifier Agent

После каждой генерации Claude запускает субагента-проверяющего: открывает дизайн во фрейме, делает скриншоты, ловит вёрстку. Автоматический QA.

05

Контекст из других проектов + Web Capture

Claude может читать файлы из других ваших дизайн-проектов без копирования. Web Capture по URL извлекает живые компоненты, не скриншот. Прототип соответствует реальному продукту.

Секреты 6–9 + Anti-slop

Оставшиеся секреты и список запретов, чтобы результат не выглядел как у всех остальных AI-генераторов.

Секреты 6–9

  • Copyright protector: «скопируй Notion» → отказ. Работает только если email-домен соответствует.
  • Snip Tool: в длинных сессиях Claude сам чистит старый контекст — качество не падает.
  • Figma Import: не копия, а воспроизведение стиля в HTML.
  • Canva partnership: официальный экспорт в полностью редактируемый файл.

Anti-slop — что запрещено

  • Агрессивные градиенты
  • Эмодзи (если не в дизайн-системе)
  • Скруглённые углы с цветным border-left (классический AI-card)
  • SVG-иллюстрации
  • Шрифты Inter, Roboto, Arial
  • Выдуманные цвета вне бренда
  • Filler-контент ради заполнения

Вот почему Claude Design визуально не похож на Lovable / Bolt / v0 / Replit. Anthropic прописали запреты в промпте осознанно. Поэтому результат отличается от других генераторов.

Плюсы и минусы честно

Чтобы принять решение, не опираясь на хайп.

Плюсы

  • Скорость итерации: десятки вариантов за минуты
  • Интерактивный результат, не картинка
  • Handoff в Claude Code в один клик
  • Экспорт в Canva редактируемый
  • Дизайн-система одна — все будущие проекты в стиле
  • Мультиплеер-режим для команды
  • Расширенный Plan Mode (10+ вопросов)
  • Мобильный дизайн часто лучше десктопного

Минусы

  • Недельная квота — очень жёсткая
  • 3–5 серьёзных проектов в неделю — потолок
  • Только веб — не в Claude Code, не в десктопе
  • Не для профессиональных дизайнеров: нет auto-layout, компонентов, pixel perfect
  • Экспорт в Figma невозможен (только из Figma сюда)
  • Без референсов результат скучный и шаблонный
  • Research preview — бета, всё может поменяться
  • Нет аудит-логов даже на Enterprise

Вердикт после разбора

После утёкшего системного промпта, тестов и потраченных $20 — пять пунктов, в которых суть.

  • Claude Design — не убийца Figma. Хайп преувеличен. Акции Figma падали задолго до релиза.
  • Это инструмент для не-дизайнеров, которые знают, чего хотят, но не умеют рисовать.
  • Контекст решает всё. Без референсов, дизайн-системы и плана результат шаблонный.
