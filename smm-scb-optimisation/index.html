Проблемы текущей модели

Дублирование ролей

3 арт-директора, 3 команды креаторов — одни и те же функции повторяются в каждом направлении

Изолированные команды

Нет обмена ресурсами между направлениями — при пиках у одной команды, другие могут простаивать

Ручные ресайзы

Значительная часть часов уходит на механические адаптации форматов, которые можно автоматизировать

Нет единого контент-плана

Работа «от задачи к задаче» без системного планирования приводит к авралам и неравномерной загрузке

План перехода

Недели 1–2

Аудит

Анализ макетов за последние 3 месяца. Классификация по сложности. Определение доли ресайзов. Формирование рубрикатора.

Недели 3–6

Шаблонизация

Figma-библиотеки компонентов. 3D-ассет-библиотеки. Настройка ComfyUI-workflow. AE-шаблоны анимаций.

Недели 7–10

Пилот — b2b Халва

Берём направление с минимальным объёмом (55 макетов). Полный цикл инхаус. Замеры скорости и качества.

Недели 11–16

Масштабирование

Подключение Совкомбанка, затем Халвы. Поэтапный переход с параллельной работой агентства на подстраховке.

Далее

Оптимизация

Непрерывные замеры, доработка пайплайнов, масштабирование AI-инструментов, корректировка рубрикатора по аналитике.

Риски и как их снимаем

Деградация качества

При переходе инхаус возможно временное снижение уровня макетов

Митигация: Первые 2–3 месяца арт-директор в режиме ручного контроля каждого макета. Далее — выборочный ревью.

Перегрузка при пиках

В высокий сезон объём может превысить возможности команды из 9 человек

Митигация: Контент-план с буфером. Заготовленные шаблоны. Возможность подключения фрилансеров на пики.

Потеря экспертизы агентства

Агентство накопило опыт работы с брендами, который может быть утерян

Митигация: Этап аудита включает фиксацию всех гайдов, приёмов, ассетов. Библиотека знаний в Notion.

Время на раскачку

Шаблонизация и настройка AI-пайплайнов требуют инвестиции времени

Митигация: Параллельная работа с агентством до полной готовности. Пилот на b2b Халве — минимальный риск.
